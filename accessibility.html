Accessibility
by Design

PeerReview Hub is built on the four POUR principles of the Web Content Accessibility Guidelines 2.0 — accessibility is a guiding principle, not a checkbox.

The Four Principles

POUR in Practice

Each WCAG 2.0 principle is directly applied within PeerReview Hub with specific, measurable design decisions.

P

Perceivable

All information and UI components must be available in ways that can be perceived — invisible to none of the user's senses.

  • Alternative text on all icons and images, compatible with screen readers
  • Text-background contrast ratio of at least 4.5:1 (WCAG 2.0 SC 1.4.3)
  • Text-only mode eliminating visual elements for a simplified, high-contrast interface
  • Captions or transcripts for any multimedia content
O

Operable

Interface elements and navigation must be operable by all users, including those who cannot use a mouse.

  • Full keyboard navigation — submit and review assignments entirely without a mouse
  • Clear visual focus indicators on all interactive elements (WCAG 2.0 SC 2.4.7)
  • Countdown display on time-limited features with an option to extend the deadline
  • No keyboard traps — users can always navigate away from any component
U

Understandable

Information and operation of the interface must be understandable — readable, predictable, and error-assisted.

  • Plain language throughout — no jargon in feedback forms or instructional text
  • Error messages that name the cause and provide the recovery pathway (SC 3.3.1)
  • Navigation appears in the same location on every page (no surprises)
  • No automatic page redirects unless explicitly triggered by the user
R

Robust

Content must be reliably interpreted by a wide range of user agents, including current and future assistive technologies.

  • Semantic HTML5 elements — headings, form labels, landmark regions
  • ARIA attributes where native semantics don't convey enough meaning
  • Automated accessibility audits using WAVE and Axe throughout development
  • Valid, well-structured markup that assistive technologies can parse reliably
